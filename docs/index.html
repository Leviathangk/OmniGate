核心特性

不是做又一个，
而是做更易用的一个

打破传统 CLI 工具硬编码 API Key 的繁琐体验，用更内聚、更可靠的方式连接每一个大模型。

01

统一供应商管理

集中式纳管多平台大模型供应商，一处配置、多 CLI 共享流量。告别在各配置文件中来回粘贴密钥的低效操作。

02

双模式运行

直连模式配置直写客户端，网关零介入；接管模式强力代理，提供路由与高阶调度。想怎么用，随你便。

03

多级接管调度

指定顺序用废自切、手动锁定单一渠道、随机分配——三种策略覆盖不同场景的 Context 缓存诉求。

04

大模型自由映射

Claude 协议别名自由转换，全局默认接管模型设置。客户端无需频繁修改模型名，网关自动转换分流。

05

全局 Prompt 管理

集中管理全局 System Prompt，可动态启用，自动注入到每次 API 通信，一键获得专属业务背景对齐。

06

主流 CLI 全面覆盖

深度适配 Claude CLI、Codex CLI、Opencode CLI 三大头部开发者编程工具，一键接管，即开即用。

双模式

直连零延迟，接管全掌控

根据你的场景灵活切换——OmniGate 既是配置生成器，也是智能代理网关。

Direct

直连模式

直接将供应商的原始连接信息与 API Key 写入 CLI 配置文件。OmniGate 作为静态配置生成器，写入后不参与任何流量转发。

  • ✓ 完全零延迟直连
  • ✓ 网关不参与通信
  • ✓ 配置一键写入 CLI
  • ✓ 适合稳定单一渠道
Proxy

接管模式

将 OmniGate 的本地网关连接信息写入 CLI 配置，所有请求通过代理进行智能转发与多路调度。

  • ✓ 多供应商智能路由
  • ✓ 用废自动切换备用
  • ✓ 熔断降级与自愈
  • ✓ 全局 Prompt 注入
容灾机制

工业级熔断，
一击必愈

应对 API 瞬时抖动与服务商封锁，OmniGate 在内存态实现高性能熔断降级系统。

01

连续异常降级

某供应商在一轮重试中全部失败，系统对其施加惩罚：顺序模式下优先级延后 1000 位；随机模式下流量权重指数衰减，直至最低权重 1。

02

一击必愈 · Self-Healing

被惩罚节点只要成功响应一次，立刻秒级洗刷所有惩罚评分，满血复活。无需人工干预，无需等待冷却。

03

大家平权 · Equal Reset

若所有节点均处于惩罚状态，系统自动清空所有惩罚记录，重回同一起跑线平权竞争，完美规避全网死锁。

架构优势

  • 内存安全 Rust 所有权机制，编译期零成本抽象，运行时零 GC 暂停
  • 多路复用 Reqwest 连接池复用，单连接并发多请求，代理延迟 <1ms
  • 零磁耗 RwLock<HashMap> 内存态跟踪，杜绝 SQLite 高频磁盘 IO
  • 原生体验 Tauri 轻量桌面壳，安装包极小，内存占用远低于 Electron

运行指标

  • 代理延迟 <1ms（本地网关转发）
  • 熔断响应 毫秒级内存状态切换
  • 自愈速度 1 次成功请求即刻恢复
  • 磁盘写入 仅配置文件，运行时零 IO
